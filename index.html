博客搭建

在经历了一天的努力，也终于是搭建好自己的博客，搭建过程中也踩了许多的坑，查询了网上大佬的资料和自己同学已经搭建好的博客，还有up主的视频讲解（视频底下的评论区大佬也很多），最终将自己的博客搭建起来了！

Hexo 搭建步骤

1.Git的安装

安装Git主要为了方便管理Hexo和上传。

Tip：使用cmd也是可以的（奈何博主是废物，学的不太会）！

windows：到Git官网就可以下载。

Linux：（博主还没学，从隔壁那个好同学那里摘过来的命令）

安装好后“win+R”，打开运行输入cmd，在cmd中输入 git –version ，如果输出版本号，则正确。

Tip：往后的验证都是在 cmd 中执行。

windows：到Node.js官网就可以下载。（这边建议选择左边，因为官方建议，得到了官方认证！）

4.GitHub创建个人仓库

首先需要注册一个GitHub账户
详细教程看这里

注册登录之后，在GitHub左上角有一个New repository（也可能是其他地方），新建一个仓库。
Tip：注册完成之后需要记住你的用户名，注册邮箱和密码，接下来需要使用到。

创建一个与你用户名相同的仓库，后面加.github.io，只有这样，在后面的GitHub page的时候才会被识别，也就是yourname.github.io，这其中的 yourname 就是你的用户名，然后点击create repository。

5.生成SSH添加到GitHub

回到 git bash中（以下操作使用 cmd 同样可行）

这里的yourname同上面的yourname，也就是你的用户名，youremail也是如此。

使用下面两条可以检查有没有输入正确

然后创建SSH，这里只需要一路回车，总共是四次

这里的youremail同上

进入C盘，然后进入Users（用户）如果没有修改就是在Admin目录下的.ssh文件，找到id_rsa.pub，使用记事本或者下载一个 notepad++ （记事本更加便捷）。将id_rsa.pub里的信息进行ctrl+a全选，再ctrl+c复制。

再回到你的Github，找到setting（点击你同学的倒三角），找到SSH key的设置选项，再点击New SSH key把你的id_rsa.pub里面的信息复制进去。

然后回到git bash（或者cmd中）查看是否成功

输出了successfully authenticated，就代表着成功绑定。

做到这里已经成功了一大半了，现在就差将hexo部署到GitHub

最后

如果hexo d后看见Deploy done:git就代表部署成功

注:deploy时可能要你输入username和password，由于GitHub规则更新这里的密码不是你的账户密码，而是令牌。进入github->Developer settings->personal access tokens->generate new token，note可以随便填，其他的可以全部勾上，然后generate token来创建一个新的令牌，这个令牌建议截图或者复制保存下来，因为他只会出现一次（作者本人没出现这个要密码的情况，那是最好的少麻烦一点）。

过一会儿就可以在http://yourname.github.io 这个网站看到你的博客了。
